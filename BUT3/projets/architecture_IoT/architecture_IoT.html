Architectures Réseaux
Hybrides pour l'IoT

Conception, mise en œuvre et comparaison de solutions de communication sans fil (Bluetooth, BLE, LoRa) pour un environnement de capteurs connectés.

ESP32 C++ / Arduino ThingSpeak Électronique

Contexte & Objectifs

L'objectif principal était de concevoir un système complet de collecte de données environnementales. Le projet repose sur l'utilisation de plusieurs technologies de transmission intégrées à des microcontrôleurs ESP32.

Fonctionnalités de l'architecture :

  • Collecte : Température via capteur KY-013 (Thermistance NTC).
  • Alerte : Signal sonore via buzzer KY-012 si seuil dépassé.
  • Transmission : Hybride (Bluetooth, BLE, LoRa).
  • Visualisation : Locale (OLED) et Cloud (ThingSpeak).

Retour d'expérience : Le problème "NAN"

Dysfonctionnement Capteur KY-013

Lors de l’utilisation d’un code de base sur la carte Heltec, la console affichait NAN (Not A Number). Ce dysfonctionnement rendait impossible l'exploitation des températures.

Solution mise en œuvre :

  • Utilisation de la valeur absolue de la résistance avant le calcul logarithmique.
  • Ajustement de la formule de conversion (Steinhart-Hart) pour la cohérence physique.

Le diagnostic complet et les formules mathématiques corrigées sont disponibles dans le rapport ci-dessous.
